# Documentation guidelines

All files under `docs/` are human-facing references for the ADHD Productivity platform.

- Write in Markdown with clear headings and tables of contents when documents exceed one screen.
- Link to source files or modules using relative paths (e.g., `../app/services/...`).
- Keep terminology consistent with the five epics defined in the root `README.md`.
- When documenting API changes, include request/response examples plus references to the relevant
  Pydantic schemas.
- Do not duplicate content across multiple docs; instead, cross-link existing sections.
